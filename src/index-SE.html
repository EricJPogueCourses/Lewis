Software Engineering

Sprint 1: Getting Started, Software Engineering Methods, Software as a Service, and Tools of the Trade

Welcome to Software Engineering! By this point in your software development journey I'm sure that you are confident in writing moderately complicated software programs. You've learned about structured programming techniques like sequence, selection, and repetition. You've learned how to break complicated programs down into more manageable pieces by using functions. You've even understand Object-Oriented Programming can help you develope more complex applications by utilizing encapsulation, inheritance, and polymorphism.

That's wonderful, however to create sophisticated modern programs we need to master some additional skills that allow us to effectively work across teams to predictably develop, enhance, and maintain complex systems that meet real users needs. These additional Software Engineering skills are included in what we call the Software Development Life Cycle and include topics like Project Management, Architecture, Requirements, Testing, Support, and Maintenance. These are some of the topics that we will be exploring as part of this Software Engineering course.

You have likely noticed that I am using the term “sprint 1” to describe our first set of course activities and assignments. Sprints are part of the Scrum development process. The term is used to describe a unit of work which will be delivered in a given amount of time. We will be learning more about sprints, Scrum, and Agile development in the coming days. For now it is sufficient to know that our sprint 1 activities listed below are expected to take approximately 20 hours of focused effort to complete.

Let's start with what we hope to learn in sprint 1. By the end of sprint 1 we will be able to:

  • describe the basic steps in the Software Development Life Cycle (SDLC)
  • compare and contrast software life cycle methodologies
  • explain why agile development has become the preferred software development process
  • define software quality
  • describe software as a service and its benefits
  • explain legacy code
  • list the pros and cons of cloud computing
  • describe client-server, three-tier and MVC design patterns
